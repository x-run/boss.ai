# PRD — boss.ai プロダクト要件定義書

> 最終更新: 2025-02 / Phase: MVP (デモ・営業用)

## 1. プロダクトビジョン

**AIがディレクションし、人間がクラフトする動画制作プラットフォーム。**

依頼主は AI と対話して要件を固めるだけ。
AI がタスク分解・クリエイター選定・進捗管理・一次品質評価を自動化し、
人間の編集者は「クリエイティブ」という最も人間的な仕事に集中する。

```
Client（依頼主）
   ↓ ブリーフ作成（AIチャット）
AI Director
   ↓ タスク分解 → Worker 選定 → 発注
Worker（編集者）
   ↓ 編集・納品
AI Review
   ↓ 一次品質チェック
Client
   ↓ 最終確認 → 承認
Done
```

## 2. ターゲットユーザー

### Client（依頼主）
- SNSマーケティング担当者、D2Cブランド運営者
- 動画制作を外注したいが、ディレクションに時間を取られている
- 月に数本〜数十本の動画を継続的に必要としている

### Worker（編集者）
- フリーランスの動画編集者
- TikTok / Reels / YouTube ショート / 広告動画に強い
- Premiere Pro, CapCut, DaVinci Resolve などを使用
- 安定した案件供給を求めている

## 4. フェーズ計画

### Phase 0: MVP（現在）— デモ・営業用
**目的**: 投資家・顧客候補にコンセプトとUXを見せる

- [x] LP（ビジョン説明 + CTA）
- [x] ブリーフ作成（AIチャット風UI、ローカル保存）
- [x] Job管理（一覧・詳細・ステータス管理）
- [x] Worker登録（4ステップ式、ローカル保存）
- [x] Worker一覧・詳細
- [ ] LP → Worker登録 → 一覧 の導線整理
- [ ] デモ用サンプルデータの自動生成

**データ永続化**: localStorage のみ（バックエンド不要）

### Phase 1: v1 — プロトタイプ
**目的**: 少数のClient + Workerで実運用テスト

- [ ] バックエンド導入（認証、DB、API）
- [ ] AI Director: タスク分解 + Worker マッチング
- [ ] AI Review: 納品物の一次品質評価
- [ ] 案件の発注→受諾→制作→納品→レビューフロー
- [ ] 通知機能（メール or Slack）
- [ ] Worker ダッシュボード

### Phase 2: v2 — スケール
**目的**: 本数増加に耐えるプラットフォーム

- [ ] AI による自動フィードバック + 修正サイクル
- [ ] 決済・請求機能
- [ ] Worker 評価・レーティング
- [ ] 案件テンプレート
- [ ] アナリティクス（制作リードタイム、品質スコア推移）

## 5. 非機能要件

| 項目 | MVP | v1 |
|------|-----|-----|
| 認証 | なし | メール + OAuth |
| データ永続化 | localStorage | PostgreSQL + API |
| ホスティング | Vercel (静的) | Vercel + API サーバー |
| レスポンシブ | モバイル対応済み | 同左 |
| ブラウザ対応 | Chrome, Safari (最新) | 同左 + Firefox |
| パフォーマンス | — | FCP < 1.5s |

## 6. 成功指標（v1以降）

- Client: ブリーフ作成 → Job保存完了率 > 80%
- Worker: 登録完了率 > 70%
- マッチング: AI選定 → Worker受諾率 > 60%
- 品質: AI一次評価パス率 > 75%
- リードタイム: 発注 → 納品 < 48h（ショート動画）
